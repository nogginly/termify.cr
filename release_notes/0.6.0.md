# `termify.cr`

## Release 0.6.0

### IMPROVEMENTS

- Expands Termify beyond Markdown rendering.
- Add a set of ANSI terminal control helpers via functional modules:
  - `ANSI::Cursor` for cursor movement and visibility
  - `ANSI::Screen` for switching screen buffers
  - `ANSI::Clear` to clear line or screen parameterised by `ANSI::Erase` enum (`Before`, `After`, `All`)
  - `ANSI::Mouse` uses SGR extended mouse reporting (and not much else for now.)
- Add a platform-aware `Terminal` singleton.
  - `Termify.terminal` returns a lazily-initialised platform singleton
  - `color_supported?` detects ANSI color support
  - `truecolor_supported?` detects 24-bit color support
  - `with_raw_input(&)` switches `STDIN` to raw/VT mode for the duration of a block, then restores it
  - `cursor_row` queries the cursor's current row via a DSR sequence
  - `setup_console` / `restore_console` to allow for platform-dependent config for VT output mode management (no-ops on Unix)
- Add `ANSI::SubScroller` helper for constraining output to a fixed-height scroll region within the terminal, useful for streaming output.
